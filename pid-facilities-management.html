What's actually live right now: click-to-chat. Every WhatsApp number in Work Orders is a real wa.me link — clicking 💬 opens a real WhatsApp chat, pre-filled, ready to send. Fully automatic sending (no click, fires by itself) is not live and can't be turned on from here — it needs a Meta-approved WhatsApp Business API account with a permanent access token, and that token must never sit in this file (anyone viewing page source could steal it and send messages on your account/bill). Turn on real automatic email sending

1. Sign in with Microsoft is already built
Set up under Settings earlier — Azure App Service Authentication
2. In the Azure Portal → your Web App → Authentication
Click the Microsoft identity provider → App registration link to open it in Entra ID
3. API permissions → Add a permission
Microsoft Graph → Delegated → search Mail.Send → Add
4. Click "Grant admin consent"
Needed once, by an account with admin rights on the tenant
5. Sign out and sign back in here
So the new permission is included in your sign-in token

Once done, the ✉ button next to any work order's assignee sends a real email automatically — no email client opens, no click-to-send. Until then, it falls back to opening a pre-filled email in your own mail client, which still works today.

Microsoft sign-in for Admin

Checking connection…

This uses Azure's own built-in authentication (App Service "Authentication" / Easy Auth) — real Microsoft/Entra sign-in, handled entirely by Azure's servers. No OAuth code or secrets live in this app.

1. In the Azure Portal
Open your Web App → left menu → Authentication
2. Add identity provider
Choose Microsoft → Create new app registration (Quick) → keep defaults
3. Restrict access
Choose "Allow unauthenticated access" — this lets the app load for everyone; the Admin gate below is enforced inside the app itself
4. Save, then reload this app
The sign-in box in the sidebar will switch from "Not connected" to a real "Sign in with Microsoft" link

Note: this checkbox controls the in-app view switch only. Since this app has no backend, it doesn't restrict the underlying data — real per-role data security needs a backend, same as the credentials feature.

SharePoint / OneDrive file storage

Checking connection…

Reports and exported files can be saved directly to Microsoft, using the same Graph token as the email feature — no separate backend. By default it saves to a folder in your own OneDrive (which is SharePoint-backed storage). To target a real shared SharePoint site instead, paste its Site ID below.

1. Same Azure AD app registration as email
Web App → Authentication → Microsoft provider → App registration
2. API permissions → Add a permission
Microsoft Graph → Delegated → search Files.ReadWrite (or Files.ReadWrite.All for shared sites) → Add
3. Grant admin consent, then sign out and back in
Same as the email setup — the new permission needs to be in your token
